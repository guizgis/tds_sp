# 可信数据空间服务平台 (TDS-SP) 项目规范

## 1. 平台定位
本项目定位为符合国家数据基础设施 (NDI) 标准的 **可信数据空间服务平台**。作为逻辑数据空间的治理核心，平台主要服务于 **空间运营方 (Space Operator)**，承担管理、审核与监控职责。

### 核心功能定位：
*   **管理与治理中心**：平台不对接最终用户业务操作，而是作为治理侧，处理来自接入连接器的各类申请（身份、资源、空间等），并执行全生命周期管理。
*   **接入主体交互模式**：数据提供方、使用方等接入主体不直接在平台上进行业务操作，而是通过各自的 **接入连接器 (Access Connector)** 发起身份注册提交、目录发布提交等业务请求。
*   **多租户治理架构**：平台支持创建多个逻辑隔离的可信数据空间，允许不同的空间运营方在同一套基础设施上实施差异化的治理策略。

## 2. 角色职责定义
*   **平台管理员 (System Admin)**：负责平台级基础设施维护，以及对空间运营方资质、全局接入主体初始准入的最高层级审核。
*   **空间运营方 (Space Operator)**：平台的核心用户。负责创建逻辑空间，审核空间内成员（Provider/Consumer）的加入申请，以及对空间内上架的数据产品进行合规性审核。
*   **接入主体 (Participant)**：通过 **接入连接器** 与平台建立通信。利用连接器提交注册材料、发布数据目录元数据、发起合约签署请求。

## 3. 核心功能架构
*   **身份管理**：管理基于 DID 的分布式身份，处理来自连接器的注册申请。
*   **目录管理**：汇聚并审核各连接器上报的数据产品，形成全域/空间目录。
*   **空间管理**：提供逻辑空间的生命周期管理及配置功能。
*   **合约与监控**：提供数字合约存证及连接器运行状态的实时监测。
