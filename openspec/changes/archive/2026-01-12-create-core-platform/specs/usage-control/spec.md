## ADDED Requirements

### Requirement: Policy Execution Monitoring
The system SHALL link with the connector's data usage environment (Sandbox/TEE) to receive real-time execution status and ensure operations comply with contract instructions.

#### Scenario: Receive execution log
- **WHEN** a connector reports a data access event
- **THEN** the system verifies it against the active contract instructions
- **AND** logs the result

### Requirement: Evidence Collection
The system SHALL collect Proof of Usage (PoU) and fine-grained logs generated by connectors.

#### Scenario: Archive proof of usage
- **WHEN** a usage session completes
- **THEN** the system stores the cryptographic proof provided by the connector

### Requirement: Violation Handling
The system SHALL block or alert on key violation behaviors.

#### Scenario: Detect violation
- **WHEN** a connector reports access outside of permitted hours
- **THEN** the system triggers a violation alert
- **AND** marks the contract as breached if configured
