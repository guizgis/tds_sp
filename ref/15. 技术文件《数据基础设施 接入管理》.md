II


#### 数据基础设施接入管理

1 范围


本文件规定了数据基础设施建设中业务节点、接入连接器接入区域/行业功能节点过程中的技术、
管理、安全以及合规等方面要求。

本文件适用于指导各区域/行业功能节点开展接入管理，指导业务节点、接入连接器接入数据基础
设施体系。


2 规范性引用文件


下列文件中的内容通过文中的规范性引用而构成本文件必不可少的条款。其中，注日期的引用文件，
仅该日期对应的版本适用于本文件；不注日期的引用文件，其最新版本（包括所有的修改单）适用于本
文件。

3 术语和定义


下列术语和定义适用于本文件。
3.1

全域功能节点global function node
指数据基础设施中面向全域提供统一标识管理、统一身份管理、统一目录管理和运行监测等服务的
节点。

[来源：NDI-TR-2025-02 数据基础设施互联互通基本要求]
3.2

区域功能节点regional function node
指数据基础设施中面向特定区域提供身份注册和核验、数据登记、数据目录查询、数据标识解析、
运行监测等服务的节点。

[来源：NDI-TR-2025-02 数据基础设施互联互通基本要求]
3.3

行业功能节点industry function node
指数据基础设施中面向特定行业提供身份注册和核验、数据登记、数据目录查询、数据标识解析、
运行监测等服务的节点。

[来源：NDI-TR-2025-02 数据基础设施互联互通基本要求]
3.4

业务节点service node


1


指数据基础设施中各区域、行业及企业的数据流通利用平台。

[来源：NDI-TR-2025-02 数据基础设施互联互通基本要求]
3.5

接入连接器access connector
连接接入主体与接入主体或接入主体与业务节点的规范化软硬件系统，数据供需双方均可通过接入
连接器接入数据基础设施。

[来源：NDI-TR-2025-05 数据基础设施接入连接器技术要求]
3.6

管理主体 management entity
区域数据主管部门或行业主管部门指定的区域/行业功能节点运营主体，负责在本区域、本行业组
织开展业务节点、接入连接器的接入管理活动。
3.7

注册地 access location
为业务节点、接入连接器注册身份的区域/行业功能节点。


4 管理主体管理要求


4.1 管理主体


a) 区域/行业功能节点运营主体是接入管理的管理主体，负责在本区域、本行业组织开展业务节
点、接入连接器的接入管理活动。
b) 区域/行业有多家功能节点运营主体的，由各运营主体协商明确接入管理职责分工。协商不成
的，由区域数据主管部门或行业主管部门指定一家运营主体作为管理主体负责本区域、本行业
的业务节点、接入连接器接入管理活动。


4.2 管理制度


a) 管理主体应制定接入管理制度，配备专门人员进行业务节点、接入连接器的接入管理。相关接
入管理制度应在管理主体运营的区域/行业功能节点页面的显著位置予以公示。
b) 管理主体制定的接入管理制度应至少包括接入条件、接入流程、审核流程、审核时限、常态化
管理、安全要求等。接入管理制度不应与本规范以及NDI-TR-2025-02、NDI-TR-2025-03、
NDI-TR-2025-04、NDI-TR-2025-05、NDI-TR-2025-06 的要求相悖。


4.3 注册地管理


a) 业务节点、接入连接器的信息更新、服务注销以及信息上报等活动由注册地管理主体组织开展。
b) 业务节点、接入连接器可选择其所在地、主要服务领域的区域/行业功能节点作为注册地，也
可选择其他区域/行业功能节点作为注册地。管理主体不应限制业务节点、接入连接器对注册
地的自主选择。
c) 已接入、未注销的业务节点、接入连接器以相同的信息在注册地以外的其他区域/行业功能节
点申请注册的，应先在原注册地完成注销。原注册地因故无法注销的情况除外。


4.4 接入申请


a) 业务节点、接入连接器在区域/行业功能节点申请业务节点身份注册和接入连接器身份注册的，
即等同于申请接入。


2


b) 业务节点、接入连接器提交接入信息应符合NDI-TR-2025-03 的要求。
c) 业务节点、接入连接器接入应由其所属法人或其他组织提出。
d) 申请注册业务节点的法人或其他组织应在注册业务节点前完成主体身份注册。
e) 业务节点、接入连接器的地址应可通过公共互联网正常访问，如有服务网页，其页面语言应为
中文，并按相关要求完成备案。
f) 业务节点名称、业务功能简介应清晰准确、无歧义，业务功能简介可全面涵盖主要服务方向。
g) 管理主体应在业务节点接入申请之日起5 个工作日内完成接入审批。在接入审批中，如管理主
体认为有必要对业务节点服务能力、技术水平以及安全等相关方面进行深入核查的，可组织领
域专家进行评审，审批周期相应延长5 个工作日。评审应邀请业务节点相关运营或服务主体参
加，结论应向参加主体公开。
h) 管理主体应在接入连接器接入申请之日起1 个工作日内完成接入审批。
i) 对于符合要求的业务节点和接入连接器，管理主体应批准其接入，按照NDI-TR-2025-03 的要
求发放身份凭证，并按照NDI-TR-2025-02 的要求向全域报送相关信息。
j) 业务节点变更服务类型、扩充服务范围的，管理主体应参照新接入申请进行审批。业务节点、
接入连接器其他信息变更的，管理主体应在申请之日起1 个工作日内完成审批。
k) 管理主体认为业务节点、接入连接器不符合接入要求的，应对接入请求予以驳回，并说明理由。


4.5 常态化管理


a) 注册地管理主体对已接入的业务节点、接入连接器开展常态化管理。注册地管理主体的常态化
管理应遵循最小必要原则，不应对业务节点、接入连接器的正常运营产生不良影响。
b) 管理主体应在其运营的区域/行业功能节点页面显著位置公布服务监督电话，宜在其服务系统
提供线上服务投诉管理的页面，并安排专人受理针对业务节点、接入连接器的服务投诉。相关
投诉内容和处理结果应予以记录。


4.6 接入注销


a) 因故需要终止服务的业务节点可申请注销，并提交服务退出方案。
b) 管理主体应审核业务节点注销的理由和服务退出方案，并对服务退出过程进行必要的监督，保
障业务节点运营期间所涉及的业务和数据得到妥善处理。
c) 业务节点完成服务终止流程后，管理主体可对业务节点身份进行注销，注销过程应符合
NDI-TR-2025-03 的相关要求。注销后管理主体应按照NDI-TR-2025-02 向全域功能节点报送相
关信息。
d) 业务节点注销后，管理主体应保存业务节点接入期间上报的各类运行和业务数据。保存期间不
短于3 年。
e) 业务节点、接入连接器在一个注册地注销后，可在另一个注册地重新注册，也可在原注册地重
新注册。在原注册地注册的，应视为新注册，原有信息不予继承。


5 业务节点、接入连接器接入要求


5.1 服务要求


a) 业务节点应在境内设置服务系统，具有与所从事服务相适应的服务能力。
b) 业务节点应具有健全的用户个人信息保护能力、网络与信息安全保障措施，包括管理人员、网
络与信息安全管理制度、安全保障技术、应急处置预案和管理措施等。


3


c) 业务节点、接入连接器的系统和服务应符合NDI-TR-2025-02 、NDI-TR-2025-03 、
NDI-TR-2025-04、NDI-TR-2025-05、NDI-TR-2025-06 的相关要求。
d) 业务节点服务应与功能类型、服务简介相关内容一致。
e) 业务节点、接入连接器接入前应通过第三方机构对系统和服务的功能完备性、互联互通能力、
安全防护水平等进行测试或认证。
f) 业务节点、接入连接器接入后发生信息变更的，应及时进行更新。


5.2 信息公示


a) 业务节点、接入连接器接入数据基础设施后，应在其服务页面的显著位置公示接入状态，注册
地信息以及注册地服务监督电话。
b) 涉及数据存储的业务节点、接入连接器，应在其服务页面的显著位置公示数据存储相关安全策
略、保护等级等信息。
c) 业务节点因升级维护等需要暂停服务的，应提前5 个工作日在其服务页面的显著位置公示服务
暂停信息，包括但不限于暂停原因、暂停开始时间、预计暂停结束时间和受影响的范围。
d) 业务节点因不可抗力或其他无法预料的突发故障导致服务中断的，应尽快发布初步告知并在服
务中断的24 小时内公告服务中断原因、服务中断时间、预计服务恢复时间和受影响的范围。


5.3 业务管理


a) 业务节点应对其平台上的第三方数据和服务进行必要的审核和管理。
b) 业务节点应在页面显著位置公布服务监督电话，并安排专人受理针对数据和服务的投诉。相关
投诉内容和处理结果应予以记录。
c) 业务节点提供的服务，应明确服务价格或计价方式。


5.4 信息上报


业务节点、接入连接器应按照NDI-TR-2025-02的要求向注册地区域/行业功能节点报送运行和业务
信息。


5.5 服务退出


因故需要终止服务的业务节点，应制定服务退出方案，妥善处理运营期间所涉及的业务和数据。服
务退出方案应包括但不限于：数据迁移、数据销毁或删除、相关数据备份的保存方式及期限、用户通知
流程以及退出过程中用户业务过渡、数据隐私和安全保护措施等。
