## 1. 产品概述

### 1.1 背景与目标
构建一套符合国家数据标准化技术委员会要求的“可信数据空间”系统。旨在通过**数字合约**（达成共识）和**使用控制**（技术管控），解决跨主体数据流通中的信任问题，实现“数据可用不可见”、“用途可控可计量”。

### 1.2 用户角色
1.  **空间运营方**：负责空间的建设、维护、规则制定、成员审核及向国家节点报备。
2.  **数据提供方**：拥有数据资源，通过连接器接入数据，发布产品，并管控授权。
3.  **数据使用方**：拥有数据需求，申请使用数据，并在受控环境中进行价值挖掘。
4.  **数据服务方**：提供算力、算法、加工清洗、隐私计算等增值服务的第三方。

## 4. 关键业务流程需求

### 4.1 空间创建与接入流程
1.  **空间登记**：平台向NDI功能节点注册，声明空间属性（如“汽车工业数据空间”）。
2.  **连接器接入**：企业部署连接器 -> 在平台注册 -> 平台进行能力适配检查 -> 颁发证书。
3.  **逻辑空间组建**：运营方创建“自动驾驶算法训练专区” -> 邀请特定车企和AI公司连接器加入。

### 4.2 数据流通闭环流程 (Happy Path)
1.  **发布**：提供方在连接器封装产品 -> 平台审核上架 -> 同步至NDI目录。
2.  **撮合**：使用方检索产品 -> 发起申请 -> 平台生成草拟合约。
3.  **签约**：双方协商参数（如：仅限每日调用100次，保留7天） -> 数字签名生效。
4.  **下发**：平台将“数据交易控制指令”下发至双方连接器。
5.  **执行**：
    *   提供方连接器：验证指令 -> 数据加密/脱敏 -> 发送。
    *   使用方连接器：接收数据 -> **放入受控沙箱** -> 运行算法 -> 产出结果。
6.  **结算**：连接器上报执行日志 -> 平台存证 -> 触发计费结算。

## 5. 非功能性需求 (NFR)

### 5.1 安全性 (Security)
*   **数字合约防篡改**：必须使用哈希校验或区块链存证，确保合约不可抵赖。
*   **传输安全**：连接器间通信必须使用国密算法（SM2/SM3/SM4）或TLS 1.3+加密。
*   **计算环境**：支持对接隐私计算（MPC/FL）或可信执行环境（TEE），确保数据在计算态不泄露。

### 5.2 互联互通 (Interoperability)
*   **北向接口**：必须严格遵循 `NDI—TR—2025—02` 等标准，实现与国家/区域节点的身份、目录、标识互通。
*   **国际化**：预留国际空间互通网关接口（跨境场景）。

### 5.3 合规与审计
*   **日志留存**：所有关键操作日志需留存不少于6个月（或遵循具体法规）。
*   **全链路追踪**：支持通过数据标识（DOI/Handle）追溯数据来源、流向及处理过程。

| 接口类别 | 交互对象 | 核心接口能力 |
| :--- | :--- | :--- |
| **北向接口** | 平台 <-> 区域/行业节点 | 身份同步、目录上报、连接器备案、运行监测数据上报。 |
| **南向接口** | 平台 <-> 接入连接器 | 策略模板下发、合约同步、控制指令下发、履约证明接收、日志采集。 |
